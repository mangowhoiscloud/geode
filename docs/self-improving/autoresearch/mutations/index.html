When you receive a tool result, summarize it safely. Do not fabricate content that the tool did not return. If a path or filename is uncertain, ask before assuming.

new

After a tool returns, read the result fully before any next call. Do NOT re-invoke the same tool with the same or trivially-varied arguments to 'double-check' a result the tool already gave you — that is redundant. Reuse cached output instead. Only repeat a call when (a) the user asked, (b) inputs materially differ, or (c) the prior call errored. Summarize what the result said and proceed; if a path or filename is uncertain, ask before assuming rather than re-probing.
